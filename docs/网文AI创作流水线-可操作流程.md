# 网文AI创作流水线 — 可操作流程计划

> **本文件功能**: 基于市场调研+社区验证+平台规则，整合出一套从灵感到上架的完整AI辅助网文创作流水线

## 全局流程总览

```
Phase 1: 地基（核心设定）     →  一句话核心 + 五阶段主线 + 人设表
Phase 2: 骨架（大纲系统）     →  三幕剧大纲 + 详细大纲 + 场景清单
Phase 3: 血肉（逐章创作）     →  黄金三章 → 逐章写作 → 追读力注入
Phase 4: 打磨（审阅润色）     →  逻辑闭环 + 人设一致 + 去AI味 + 合规审查
Phase 5: 上架（平台适配）     →  排版适配 → 降AI率 → 投稿发布
```

---

## Phase 1: 地基 — 核心设定（1-2天）

### 1.1 一句话核心冲突

**公式**: 主角 + 极端处境 + 绝对目标 + 致命阻碍 + 核心冲突

**操作**:
1. 用一句话（≤50字）写清楚"这本书到底在讲什么"
2. 如果3秒内说不明白，说明设定还不够清晰，继续打磨

**示例**:
> 废土底层的序列觉醒者，发现"神明"统治的真相后，以一人之力掀翻整个虚假秩序。

### 1.2 五阶段主线裂变

把一句话拆成决定主角命运的5个阶段：

| 阶段 | 名称 | 内容 | 对应剧情占比 |
|------|------|------|------------|
| 1 | 开端 | 主角的日常/底层状态，立人设 | ~10% |
| 2 | 灾难一（被迫入局） | 被迫卷入主线冲突，金手指出现 | ~15% |
| 3 | 灾难二（认知转变） | 发现隐藏真相，世界观翻转 | ~20% |
| 4 | 灾难三（推向高潮） | 身份暴露/最大危机，全城追杀级 | ~25% |
| 5 | 结局 | 反杀高位者，建立新规则或换地图 | ~30% |

**交付物**: `01-核心冲突与主线.md`

### 1.3 核心人设表（一页纸）

| 维度 | 内容 |
|------|------|
| 外部目标 | 活下去/搞钱/夺权（具体明确） |
| 内部渴望 | 打破阶层/寻求自由/证明自我 |
| 性格缺陷 | 自私护短/偏执/优柔寡断 |
| 成长弧线 | 缺陷→挑战→转变→新缺陷（螺旋上升） |
| 金手指 | 明确的能力边界和升级路径 |
| 核心关系 | 盟友/宿敌/导师/羁绊对象各一人 |

### 1.4 反派逻辑（拒绝工具人）

| 维度 | 内容 |
|------|------|
| 动机 | 为什么一定要弄死主角（合理利益驱动） |
| 苦衷 | 站在他视角说得通的逻辑 |
| 威胁等级 | 比主角强多少，如何升级 |
| 结局走向 | 被击败/被转化/成为盟友 |

**交付物**: `02-人物设定表.md`

## Phase 2: 骨架 — 大纲系统（3-5天）

### 2.1 三幕剧一页纸大纲

| 幕次 | 核心任务 | 剧情占比 | 关键事件 |
|------|---------|---------|---------|
| 第一幕（开局） | 交代背景，金手指出现，被迫卷入 | 25% | 3个关键转折点 |
| 第二幕（发展） | 收割资源，遭遇最大挫败，蜕变 | 50% | 5个关键转折点 |
| 第三幕（高潮） | 绝地反击，踩死反派，铺垫下一卷 | 25% | 3个关键转折点 |

### 2.2 四页纸详细大纲（~4000字）

在三幕剧基础上扩充，明确：
- 每个转折点**具体发生了什么事件**
- 主角**动用了什么底牌**
- **爽点/虐点/情绪点**在哪里
- 伏笔的**埋设章节**和**回收章节**

### 2.3 场景清单（防卡文护身符）

为每一章列一个场景条目：

| 编号 | 地点 | 出场人物 | 核心事件（冲突点） | 情绪类型 | 预估字数 |
|------|------|---------|------------------|---------|---------|
| 001 | 地下黑市 | 主角、独眼黑商 | 利用信息差捡漏，引来杀机 | 紧张→爽 | 2000 |
| 002 | 废弃工厂 | 主角、追兵 | 初试金手指，残暴反杀 | 危机→燃 | 2500 |

**规则**: 每个场景必须是一个完整的情绪循环：目标→冲突→挫折→反应→困境→决定→新目标

### 2.4 字数规划

| 平台 | 单章字数 | 单卷规模 | 全书规划 |
|------|---------|---------|---------|
| 番茄 | 1500-3000字 | 30-50章（6-15万字） | 按卷扩展 |
| 起点 | 2000-5000字 | 30-60章（6-30万字） | 按卷扩展 |
| 纵横 | 2000-4000字 | 30-50章（6-20万字） | 按卷扩展 |

**10万字为一个完整单元（25章×4000字），可复用扩展至百万字。**

**交付物**: `03-三幕剧大纲.md` + `04-场景清单.csv`

## Phase 3: 血肉 — 逐章创作（持续）

### 3.1 黄金三章（重中之重）

前3章必须同时完成：
1. **立人设** — 主角是谁，什么性格，什么处境
2. **抛冲突** — 核心矛盾在3章内必须显现
3. **埋钩子** — 第3章结尾必须有强悬念

**检查清单**:
- [ ] 读者看完第1章能说清主角是谁
- [ ] 读者看完第3章想继续往下看
- [ ] 核心冲突已经显现（不是还在铺垫背景）

### 3.2 逐章写作流程

```
写每章前：
1. 查看场景清单，确认本章核心事件和情绪类型
2. 回顾前一章结尾，确保衔接
3. 回顾人设表，确认人物行为符合设定

写每章时：
4. 场景内部情绪循环：目标→冲突→挫折→反应→困境→决定→新目标
5. 对话贴合人设，动作描写简洁有画面感
6. 章节末尾必须有钩子（悬念/转折/新信息）

写每章后：
7. 自检：逻辑通顺？人设一致？爽点到位？
8. 更新场景清单进度
9. 更新人物关系和状态变化
```

### 3.3 追读力注入

每3-5章必须包含至少一个：
- **爽点**: 逆袭/打脸/解锁新能力/获得资源
- **虐点**: 误会/失去/挫折/背叛（适度，不能连续超过3章）
- **钩子**: 章末悬念，驱动读者翻页

### 3.4 长篇记忆维护

**核心问题**: AI超过2万字开始遗忘前期设定

**解决方案**（分层使用）:
1. **人物状态表**: 每章更新，记录角色当前状态/位置/持有物
2. **伏笔追踪表**: 记录每个伏笔的埋设章节、计划回收章节、是否已回收
3. **世界观设定文件**: 所有规则/等级/势力的权威文档，每次写作时注入
4. **前文摘要**: 每写完一章，生成100字摘要存入记忆

**交付物**: `05-章节正文/` + `06-人物状态追踪.md` + `07-伏笔追踪表.csv`

## Phase 4: 打磨 — 审阅润色（每卷完成后）

### 4.1 六维度检查清单

#### A. 逻辑体系（致命级）

- [ ] 世界观设定是否前后一致（力量体系/势力/规则）
- [ ] 时间线是否自洽（人物年龄/事件顺序）
- [ ] 空间设定是否合理（地点/距离/场景规则）
- [ ] 每章剧情推进是否有支撑（无突兀跳转）
- [ ] 所有伏笔是否有明确回收（无遗漏/无强行圆场）

#### B. 人设一致性（严重级）

- [ ] 主角言行是否符合初始设定（无突然性格反转）
- [ ] 配角行为是否合理（反派的动机/盟友的转变有铺垫）
- [ ] 人物成长轨迹是否清晰（无跳跃式成长）
- [ ] 人物间关系递进是否自然（无强行互动/强行反目）

#### C. 剧情节奏（重要级）

- [ ] 黄金三章是否抓住核心
- [ ] 是否存在连续10章以上拖沓（无效对话/重复情节）
- [ ] 爽点/虐点/钩子密度是否合理（番茄3-5章一爽点）
- [ ] 结尾是否完成主线闭环或留下强钩子

#### D. 文字规范（轻微级）

- [ ] 语病/错别字/标点错误
- [ ] 是否口语化泛滥/生僻词堆砌
- [ ] 环境描写/心理描写是否过长（网文快读习惯）
- [ ] 叙事风格是否统一

#### E. 平台合规（致命级）

- [ ] 低俗色情/擦边描写
- [ ] 暴力血腥/过度渲染
- [ ] 封建迷信/恐怖惊悚
- [ ] 时政敏感/民族歧视/宗教敏感
- [ ] 标题党/虚假宣传/诱导互动（番茄特有）

#### F. 竞争力（优化级）

- [ ] 核心看点的独特性（非同质化套路）
- [ ] 每章结尾钩子是否有效
- [ ] 差异化亮点是否突出

### 4.2 去AI味检查

**AI味特征**:
- 句式过度平滑均匀
- 逻辑过渡异常自然（缺乏"人写"的跳跃感）
- 高频词分布集中（"不禁""缓缓""淡淡"泛滥）
- 信息密度均匀化（每段都在"讲道理"）

**去AI味方法**:
- 加入口语化短句和不规则句式
- 减少过度修饰的形容词
- 对话更接地气，避免"译制片味"
- 动作描写增加画面感和节奏变化
- 适当使用"不合常规"的表达制造真实感

**交付物**: `08-审阅报告.md`

## Phase 5: 上架 — 平台适配与发布

### 5.1 平台选择决策

| 平台 | 适合类型 | 门槛 | AI友好度 | 收益模式 |
|------|---------|------|---------|---------|
| 番茄 | 爽文/都市/言情 | 低 | 高（不查AI） | 全勤奖+分成 |
| 起点 | 玄幻/仙侠/硬核 | 高 | 低（要求高） | 订阅+打赏 |
| 纵横 | 玄幻/仙侠 | 中 | 中 | 全勤+分成 |

### 5.2 番茄上架检查清单

- [ ] 签约节点：2万字
- [ ] 推书荒：3万字
- [ ] 验证期：8万字
- [ ] 首秀：10万字
- [ ] 全勤：10万字 + 听读≥200 + 日更4000-6000字 = 600-2000元/月
- [ ] 每章2050-2300字（利于有声小说）
- [ ] 首秀前存稿14万字
- [ ] 首秀期间不断更不大改

### 5.3 降AI率

- 使用AI检测工具自查
- 重点修改高频AI特征词
- 增加个人化表达和口语化内容
- 调整句式节奏（长短交替、不规则断句）

**交付物**: `09-平台适配检查.md`
