# 人物设计指南

> **本文件功能**: Phase 2b 人物设计阶段的完整指南。核心原则：人物丰满多样，不全围着主角转。

---

## 设计原则

1. **每个人物都有自己的生活在过** — 不出场的时候他们也在做自己的事
2. **每个人都有目标** — 不只是服务于主角剧情的工具人
3. **每个人都有弱点** — 完美的人物不真实
4. **每个人的说话方式不同** — 同一句话 5 个人有 5 种说法
5. **NPC 也要有个性** — 路边摊贩、酒馆老板、巡逻兵，每个人都有独特之处

---

## 主角设计

### 必须定义

| 维度 | 问题 | 注意 |
|------|------|------|
| 出身 | 从哪来？家庭背景？ | 影响行为逻辑和价值观 |
| 外部目标 | 具体要做什么？（活下去/报仇/找到XX/成为XX） | 必须具体，不能"变强"就完了 |
| 内部渴望 | 深层想要什么？（自由/被认可/安全感/归属） | 可以本人都没意识到 |
| 性格 | 怎么形容他？不是"善良勇敢"这种空话 | 用行为定义，不用形容词 |
| 缺陷 | 最大毛病是什么？ | 推动人物弧光的核心 |
| 习惯 | 有什么口头禅/小动作/偏好？ | 增加辨识度 |
| 底线 | 绝对不会做什么？ | 定义人物的天花板 |
| 禁忌 | 一碰就炸的点？ | 制造冲突的利器 |

### 主角不能做的事

- ❌ 开始就无敌（没成长空间）
- ❌ 毫无理由地圣母（帮助陌生人不需要理由）
- ❌ 女性角色倒贴（不需要理由就喜欢主角）
- ❌ 所有人都喜欢他（不真实）
- ❌ 永远正确不犯错（无聊）

## 核心配角设计（至少 3 人）

### 配角类型

| 类型 | 功能 | 设计要点 |
|------|------|---------|
| **导师/引路人** | 带主角入门，传授知识 | 有自己的过去和遗憾，不是专职解说员 |
| **搭档/兄弟** | 陪伴主角，制造互动 | 有自己的目标和故事线，不只是附庸 |
| **对手/竞争者** | 激发主角成长 | 有自己的理由，可能最后成为盟友 |
| **羁绊/恋人** | 情感支点 | 有自己的生活，不是等待主角拯救的公主 |
| **信息源** | 提供消息和情报 | 有自己的立场和目的，不是免费百科 |

### 配角独立性检查

每个配角回答以下问题（答不上来说明是工具人）：
1. 他/她自己的目标是什么？（与主角无关的目标）
2. 如果主角不出现，他/她会在做什么？
3. 他/她有什么不为人知的秘密？
4. 他/她最害怕什么？
5. 他/她和主角有利益冲突吗？

## 反派设计

### 反派不是"坏人"，是"对手"

好的反派要让读者觉得"如果我是他，我也可能这么做"。

### 必须定义

| 维度 | 问题 |
|------|------|
| 目标 | 他想要什么？（利益/权力/生存/信仰/复仇） |
| 理由 | 为什么这个目标对他很重要？ |
| 手段 | 他用什么手段？为什么选这种手段？ |
| 底线 | 他绝对不会做什么？（有人性的反派） |
| 弱点 | 他有什么致命缺陷？ |
| 视角 | 站在他的角度看，他是对的吗？ |

### 反派层级

| 层级 | 出现时机 | 特点 |
|------|---------|------|
| 小BOSS | 前10章 | 展示主角实力的垫脚石，但要有自己的故事 |
| 中BOSS | 第一卷中后期 | 有自己的势力，对主角构成真正威胁 |
| 大BOSS | 贯穿全书 | 隐藏在幕后，逐渐揭示其全貌 |
| 隐藏BOSS | 后期反转 | 可能是主角身边的人，颠覆读者认知 |

## NPC/路人设计

### 为什么 NPC 很重要

路人的一句话可以比作者写三段设定更有用：

**反面**（空洞）:
> 这座城市是大陆上最繁华的商贸中心，汇聚了来自各地的商人和冒险者。

**正面**（侧面描写）:
> 街边卖烤肉串的大爷看了他一眼："小伙子外地来的吧？看你这身行头，刚到潜龙城？告诉你，晚上别往南区走，最近那边不太平。对了，你要是手头紧，东城的佣兵公会正在招人。"

### NPC 设计模板

```
角色: {名字或描述}
出场: 在哪一章/什么场景
功能: 提供信息/制造氛围/侧面展示世界
特点: 一句话概括他的独特之处
说话风格: {方言/口头禅/语气}
```

### 常用 NPC 场景

- **摊贩/店主**: 用物价和商品展示经济体系
- **酒馆常客**: 用八卦和传闻展示世界观
- **巡逻兵/守卫**: 用规矩和检查展示社会秩序
- **同行/竞争对手**: 用对比展示主角的水平和特点
- **老前辈/退休强者**: 用回忆展示历史和传说

## 人物关系管理

### 关系类型

```
师徒/师生  —  传承、期望、叛逆
兄弟/战友  —  信任、牺牲、分歧
恋人/暧昧  —  障碍、误解、成长
敌对/竞争  —  动机、升级、转化
利用/交易  —  利益、背叛、反转
亲属/血缘  —  羁绊、责任、冲突
```

### 关系演进

关系不是静态的，必须随剧情变化：
- 陌生人 → 熟人 → 信任 → 背叛 → ？
- 敌人 → 对手 → 尊重 → ？
- 利用 → 习惯 → 依赖 → ？

### 关系图使用

用 `memory_graph` 工具：
- `memory_graph(action="subgraph", hash={主角hash}, radius=2)` 查看主角的二度关系
- 发现孤立的角色（没关系的NPC需要补充关联）
- 发现关系过密的区域（是否所有角色都只围着主角转？）
