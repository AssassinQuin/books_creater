# 网文创作 Skill

> **本文件功能**: 网文创作全流程引导系统。用户直接聊天触发各阶段，AI 自动识别意图并执行对应工作流。

## 核心写作理念（铁律，所有阶段必须遵守）

1. **世界观严谨完整**: 种族、能力、势力、地区、货币、衣食住行——每个维度都有明确设定，不能模糊带过
2. **人物丰满多样**: 不全围着主角转。NPC 有自己的生活，反派有自己的逻辑，路人有自己的故事
3. **禁止 AI 味设定**: 不准加"使用能力的代价""反噬""寿命折损""精神崩溃"等廉价戏剧冲突
4. **日常要充实**: 路边摆摊大爷说"小伙子厉害啊，潜龙榜排第 37"，用侧面描写建立世界观，不要空洞设定堆砌
5. **全白话接地气**: 对话像真人说话，描写有画面感，拒绝文绉绉的 AI 腔

## 意图路由

用户消息匹配以下关键词时，进入对应阶段。**一次只匹配一个阶段**。

```
关键词                                          → 阶段
───────────────────────────────────────────────────────────────
"头脑风暴"/"聊聊点子"/"我有个想法"/"灵感"          → Phase 0-1: 灵感收集+头脑风暴
"建世界观"/"设计世界"/"设定世界"/"世界观"           → Phase 2: 世界观建模
"设计人物"/"加个人物"/"人物卡"/"人物"              → Phase 2: 人物设计
"生成大纲"/"做大纲"/"规划剧情"/"大纲"              → Phase 3: 大纲生成
"写第N章"/"继续写"/"开始写"/"写一章"              → Phase 4: 逐章写作
"审阅"/"检查"/"review"/"找问题"                   → Phase 5: 审阅修改
"上架"/"发布"/"番茄适配"/"起点"                   → Phase 6: 平台上架
"改设定"/"修改世界观"/"改人物"/"调整"              → 更新模式: 级联更新
"项目状态"/"当前进度"/"status"/"进度"              → 状态总览
"加素材"/"拆书"/"参考"/"借鉴"/"找参考"            → 素材库操作
```

**无匹配时**: 检查当前项目进度（memory_search），建议下一步该做什么。

## Phase 0-1: 灵感收集 + 头脑风暴

**触发**: 用户说"头脑风暴"/"我有个想法" 等关键词

**目标**: 从用户的碎片灵感中提炼核心冲突和主线方向

### 流程

1. 确认小说名（如果没有则询问）
2. 确保项目目录存在：`novels/{小说名}/`
3. **读取** `references/brainstorm-guide.md` 获取引导话术
4. 进入引导式对话，**一次只问一个问题**（不要一次列出 10 个问题吓到用户）：

   **第 1 轮** — 画面感：
   > "你最兴奋的场景是什么？描述一下那个画面。哪怕只有一句话也行。"

   **第 2 轮** — 主角特质：
   > "主角最让你着迷的特质是什么？他/她是个什么样的人？"

   **第 3 轮** — 读者情绪：
   > "你希望读者读完最强烈的情绪是什么？爽？虐？燃？感动？"

   **第 4 轮** — 反派/对立面：
   > "有没有一个你特别想写的对手/反派？他为什么跟主角作对？"

   **第 5 轮** — 独特规则：
   > "这个世界最不一样的规则是什么？什么是这里的人觉得理所当然，但读者会觉得'卧槽'的？"

   **后续轮** — 根据用户回答深入追问，挖掘更多细节

5. 每个回答**立即存入 Memory**：
   ```
   memory_store(
     content="用户灵感：{精炼后的内容}",
     metadata={tags: "project:{小说名},idea"}
   )
   ```

6. 头脑风暴结束时，**AI 整理输出**：
   - 识别出的核心冲突（1-3 个候选，让用户选）
   - 潜在主线方向（2-3 个方向供选择）
   - 亮点场景列表
   - 建议的小说类型（玄幻/都市/言情/...）

7. 用户选定后，**存储核心决策**：
   ```
   memory_store(content="核心冲突：...", metadata={tags: "project:{小说名},decision"})
   ```

8. `git add . && git commit -m "Phase 0-1: 头脑风暴完成 - {小说名}"`

## Phase 2: 世界观建模

**触发**: 用户说"建世界观"/"设定世界" 等关键词

**前置检查**: 必须已完成 Phase 0-1（Memory 中有核心冲突）

### 流程

1. 从 Memory 召回核心冲突 + 所有灵感碎片
2. **读取** `references/worldbuilding-template.md`
3. 引导用户逐维度建立世界观（**一次一个维度**，不一次全问）：

   **维度 1 — 种族与物种**
   - 这个世界有哪些种族？
   - 各种族的特征、寿命、分布？
   - 种族之间的关系（友好/敌对/从属）？

   **维度 2 — 势力与组织**
   - 主要势力有哪些？（门派/国家/公司/帮派）
   - 各势力实力对比？
   - 势力之间的关系（同盟/敌对/贸易/暗中操控）？

   **维度 3 — 地理与区域**
   - 故事发生在什么地理范围？
   - 主要区域及其特色？
   - 危险区域/禁区？
   - 从 A 地到 B 地需要多久？

   **维度 4 — 能力与等级**
   - 力量来源是什么？（修炼/觉醒/系统/血脉/科技）
   - 等级划分（几级？每级叫什么？差距多大？）
   - 晋升条件（升级需要什么？要多久？）
   - 不同体系间的克制关系
   - **注意**: 不加"代价""反噬"等 AI 味设定

   **维度 5 — 经济与货币**
   - 货币体系（什么货币？面值？）
   - 物价水平（一顿饭多少钱？一把武器多少钱？）
   - 普通人怎么赚钱？修行者怎么赚钱？
   - 贫富差距？

   **维度 6 — 日常生活**
   - 衣：不同身份的人穿什么？
   - 食：吃什么？有什么特色食物？
   - 住：不同阶层的居住条件？
   - 行：怎么出行？交通方式？
   - 社会风俗、节日、禁忌？
   - 教育体系？

4. 每个维度完成后**立即存入 Memory**：
   ```
   memory_store(
     content="{维度名}：{精炼后的完整设定}",
     metadata={tags: "project:{小说名},worldbuilding"}
   )
   ```

5. 同时**写入文件** `novels/{小说名}/世界观/{维度}.md`（人类可读可编辑）

6. 所有维度完成后，**维度交叉验证**：
   - 时间×空间：主角从 A 到 B 的时间合理吗？
   - 经济×势力：势力的财力与其行为匹配吗？
   - 能力×种族：各种族的能力差异在世界观内自洽吗？

7. `git add . && git commit -m "Phase 2: 世界观建模完成 - {小说名}"`

## Phase 2b: 人物设计

**触发**: 用户说"设计人物"/"加个人物" 等关键词

### 流程

1. **读取** `references/character-design.md`
2. 从 Memory 召回世界观设定（人物必须符合世界规则）
3. 引导设计人物，**关注以下维度**：

   **主角**:
   - 外部目标（具体明确）
   - 内部渴望（深层动机）
   - 性格特点（不是"善良勇敢"这种空洞描述）
   - 弱点/缺陷（真实的人都有毛病）
   - 金手指（在力量体系中的位置）
   - 出身背景（影响行为逻辑）

   **核心配角**（至少 3 人）:
   - 各自的目标（不全围着主角转）
   - 与主角的关系和利益绑定
   - 自己的故事线

   **反派**:
   - 合理的动机（利益/信仰/仇恨/恐惧）
   - 不是纯坏，有自己的逻辑
   - 站在他视角说得通

   **NPC/路人**:
   - 路人也有个性（摊贩/酒馆老板/巡逻兵）
   - 用侧面描写丰富世界观

4. 每个人物存入 Memory：
   ```
   memory_store(
     content="{人物名}：{完整人设描述}",
     metadata={tags: "project:{小说名},character"}
   )
   ```

5. 用 `memory_graph` 建立人物关系：
   - 存储关系描述后，查询关联图检查遗漏

6. 写入文件 `novels/{小说名}/人物/{角色名}.md`

7. `git add . && git commit -m "Phase 2b: 人物设计完成 - {小说名}"`

## Phase 3: 大纲生成

**触发**: 用户说"生成大纲"/"做大纲" 等关键词

**前置**: Phase 0-2 必须完成

### 流程

1. 从 Memory 召回：核心冲突 + 世界观 + 人物设定
2. 生成**三幕剧总纲**（一页纸）：
   - 第一幕（25%）：背景+金手指+被迫卷入
   - 第二幕（50%）：发展+最大挫败+蜕变
   - 第三幕（25%）：绝地反击+解决+铺垫

3. 引导用户确认/修改总纲

4. 生成**场景清单**（每章一个条目）：
   ```
   | 编号 | 场景 | 出场人物 | 核心冲突 | 情绪类型 | 时间点 | 地点 | 主角等级 |
   ```

5. **维度节点标注**：每章标注时间推进、空间位置、能力等级

6. **伏笔规划**：在大纲中标注伏笔的埋设章节和计划回收章节

7. 全部存入 Memory（大纲 + 伏笔 + 维度节点）

8. 写入文件 `novels/{小说名}/大纲/`

9. `git add . && git commit -m "Phase 3: 大纲生成完成 - {小说名}"`

## Phase 4: 逐章写作

**触发**: 用户说"写第N章"/"继续写" 等关键词

### 上下文自动注入（写每章前执行）

```
1. memory_search(query="最近章节摘要", tags=["project:{小说名}","chapter-summary"], limit=3)
2. memory_search(query="本章相关人物", tags=["project:{小说名}","character"], limit=10)
3. memory_search(query="未回收伏笔", tags=["project:{小说名}","foreshadow"], limit=20)
4. memory_search(query="世界观设定", tags=["project:{小说名}","worldbuilding"], limit=10)
5. memory_search(query="AI味黑名单", tags=["anti-ai-pattern"], limit=20)
6. 读取场景清单中本章条目
```

### 写作规则（写入时实时检查）

- **读取** `references/writing-style.md` 和 `references/anti-ai-patterns.md`
- 对话必须像真人说话，白话为主
- 场景描写有画面感，不是空洞设定堆砌
- 用侧面描写丰富日常（路人视角/榜单/传闻/物价/街景）
- 禁止 AI 味高频词（"不禁""缓缓""淡淡""微微"）
- 禁止"代价""反噬""寿命折损"等套路
- 章末必须有钩子

### 写作后状态更新（每章写完执行）

```
1. memory_store(章节摘要, tags=["project:{小说名}","chapter-summary"])
2. memory_store/update(人物状态变更, tags=["project:{小说名}","character"])
3. 如有新伏笔: memory_store(伏笔, tags=["project:{小说名}","foreshadow"])
4. 如有伏笔回收: memory_update(标记已回收)
5. memory_store(时间线事件, tags=["project:{小说名}","timeline"])
```

### 输出

写入文件 `novels/{小说名}/正文/第{NNN}章-{标题}.md`

`git add . && git commit -m "ch{N}: {章节标题}"`

## Phase 5: 审阅修改

**触发**: 用户说"审阅"/"检查" 等关键词

### 流程

1. **读取** `references/review-checklist.md`
2. 并行派 **3 个 Agent**：

   **Agent A — 逻辑+维度**:
   - 时间线一致性（events 有序、无矛盾）
   - 空间一致性（人物不能瞬移）
   - 能力等级一致性（不能突然变强）
   - 伏笔遗漏检查

   **Agent B — 人设+文字**:
   - 人物 OOC 检测
   - 关系合理性
   - 文字质量（语病/错别字/标点）
   - AI 味检测（对照 Memory 中的黑名单）

   **Agent C — 合规+竞争力**:
   - 平台合规（色情/暴力/敏感词）
   - 爽点密度是否合理
   - 差异化亮点评估
   - 钩子有效性

3. 汇总三份报告 → 生成 `novels/{小说名}/审阅报告/第{N}次审阅.md`
4. 问题分级：致命/严重/轻微
5. `git add . && git commit -m "Phase 5: 审阅报告 - {范围}"`

## 更新模式: 级联更新

**触发**: 用户说"改设定"/"改人物" 等关键词

### 流程

1. 确认改什么（世界观/人物/力量体系/...）
2. 修改对应设定
3. **级联检查**：
   ```
   memory_search(tags=["project:{小说名}"]) → 找到所有受影响的记忆
   ```
4. 更新受影响的 Memory 条目
5. 更新对应设定文件
6. 检查大纲中哪些章节受影响 → 标记
7. 检查已写正文中哪些章节受影响 → 列出
8. **询问用户**: 哪些已写章节需要回头改？
9. `git add . && git commit -m "更新: {修改内容描述}"`

### 流程

1. 如果用户提供具体内容 → `memory_store(content, tags=["shared","material"])`

3. 如果用户说"借鉴某个场景" → `memory_search(query="场景", tags=["shared","material"])`
4. 拆书笔记 → `memory_store(content, tags=["shared","analysis"])`

## Memory 数据模型

| type | 内容 | tags 格式 |
|------|------|-----------|
| `idea` | 灵感碎片 | project:{名},idea |
| `decision` | 核心决策 | project:{名},decision |
| `worldbuilding` | 世界观设定 | project:{名},worldbuilding |
| `character` | 人物状态 | project:{名},character |
| `foreshadow` | 伏笔 | project:{名},foreshadow |
| `chapter-summary` | 章节摘要 | project:{名},chapter-summary |
| `timeline` | 时间线事件 | project:{名},timeline |
| `learning` | 写作经验 | project:{名},learning |
| `anti-ai-pattern` | AI味黑名单（跨项目共享） | shared,anti-ai-pattern |
| `material` | 通用素材（跨项目共享） | shared,material |
| `analysis` | 拆书笔记（跨项目共享） | shared,analysis |

## Git 规范

- 每个阶段完成 → commit
- 每章写完 → commit
- 每次改设定 → commit
- 格式: `Phase N: {描述}` 或 `ch{N}: {标题}` 或 `更新: {描述}`
